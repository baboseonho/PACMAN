# Game AI Lab UI 기준

이 문서는 제공된 `ui-ux-pro-max-skill-main.zip`의 데이터와 진단 도구를 현재 고2 수업 사이트에 맞게 선별 적용한 결과다. UIUX Pro Max의 화면이나 컴포넌트를 복사하는 문서가 아니라, 수업 화면의 판단 기준이다.

## 수업 맥락

- 대상: 국제학교 고등학교 2학년
- 상황: 교사와 함께 코드를 타이핑하고 결과를 바로 확인하는 45분 수업
- 실행: 외부 라이브러리와 네트워크 없이 동작하는 HTML/CSS/JavaScript
- 핵심 흐름: 미션 이해 → 코드 작성 → 즉시 채점 → FSM·Path·Play 실험
- 우선순위: 제품 전환이나 정보 밀도가 아니라 수업 순서, 읽기 쉬움, 결과 이해

## UIUX Pro Max에서 채택한 기준

ZIP의 `ux-guidelines.csv`, `styles.csv`, `typography.csv`, `html-tailwind.csv`, `design-review.md`, `design-audit.mjs`를 확인해 다음 원칙을 적용했다.

1. 본문은 16px를 기준으로 하고 긴 설명은 65–75자 안에서 읽히게 한다.
2. 모든 주요 조작 영역은 최소 44×44px을 확보한다.
3. 키보드 포커스를 3px 고대비 외곽선으로 표시한다.
4. 활성·완료·오류 상태를 색만으로 구분하지 않고 텍스트, 선, 기호를 함께 쓴다.
5. 375·768·1024·1440·1920px에서 수평 페이지 스크롤이 없어야 한다.
6. 모션은 상태 변화를 설명할 때만 쓰고 `prefers-reduced-motion`을 존중한다.
7. 섹션 간 간격은 8px 계열의 일정한 리듬을 사용한다.
8. 현재 위치, 선택된 탭, 비활성 상태를 명확히 표시한다.
9. 의미 있는 컨트롤은 레이블과 ARIA 상태를 제공한다.
10. 그라데이션, 장식용 블러, 반복 그림자보다 여백·선·타이포그래피로 위계를 만든다.

## 의도적으로 적용하지 않은 항목

- Claymorphism과 어린이용 둥근 글꼴: 고2 알고리즘 수업의 분위기와 맞지 않는다.
- 게임용 네온·글리치·3D 효과: 코드와 알고리즘 상태보다 시선을 끈다.
- SaaS 대시보드형 카드 반복: 수업 진행 순서를 여러 개의 동급 위젯처럼 보이게 한다.
- Tailwind·shadcn 도입: 기존 순수 HTML/CSS 구조에 불필요한 의존성을 만든다.
- 외부 Google Fonts: 오프라인 수업 실행과 글꼴 로딩 안정성을 우선한다.
- 모든 권장 컴포넌트의 일괄 적용: 콘텐츠 성격에 따라 과제, 개념, 게임 보드를 다르게 표현한다.

## 시각 방향

- 기본: Flat + Paper
- 주변 UI는 밝고 차분하게 유지하고 실제 미로와 코드 편집기만 어둡게 둔다.
- 카드 대신 구분선, 배경 차이, 왼쪽 강조선을 먼저 사용한다.
- 모서리는 3px 또는 6px을 기본으로 하며 큰 원형은 캐릭터와 완료 표시처럼 의미가 있을 때만 쓴다.
- 일반 화면에서는 그림자를 쓰지 않고, 모달의 층위에만 한 번 사용한다.

| Surface | `#FFFDF8` | 과제와 실험 작업면 |
| Ink | `#14213D` | 제목, 주요 버튼 |
| Text | `#28364D` | 기본 본문 |
| Muted | `#5F6978` | 보조 설명 |
| Coral | `#D94F3D` | 현재 학습 위치, 다음 행동 |

| Violet | `#6354A5` | CHASE 상태 |

## 타이포그래피

- UI: `Segoe UI Variable`, `Aptos`, `Segoe UI`, sans-serif
- Code: `Cascadia Code`, `Consolas`, monospace
- 본문: 16px / 1.55–1.65
- 과제 설명: 15–16px
- 보조 레이블도 11px 아래로 과도하게 줄이지 않는다.
- 대문자는 PATROL·CHASE·ATTACK과 짧은 알고리즘 상태에만 사용한다.
- Python 비교 연산자는 `!=`, `<=`, `>=`, `<`, `>` 형태를 그대로 사용하며 인쇄용 수학 기호로 치환하지 않는다.

## 레이아웃

- 1241px 이상: 미션 목록 / 수업 작업면 / 실시간 실험실의 3열
- 641–1240px: 미션 목록과 수업 작업면 2열, 실험실은 수업 아래
- 640px 이하: 단일 열, 11개 미션은 3열 요약 그리드
- 380px 이하: 미션 그리드는 2열
- 본문 직접 자식은 최대 780px, 설명 문장은 최대 약 68–70ch
- 코드 편집기, 지도, 미로는 자기 영역 안에서 축소하며 페이지 가로 스크롤을 만들지 않는다.

## 컴포넌트와 상호작용

- 기본 버튼 높이 44px, 모바일과 데스크톱 모두 동일한 클릭 기준을 사용한다.
- 주 행동은 Ink 채움, 다음 미션은 Coral 채움으로 구분한다.
- 현재 미션은 왼쪽 Coral 선, 현재 실험 탭은 아래 Coral 선으로 표시한다.
- 비상 스킵은 일반 완료 체크와 구분해 노란 `SKIP` 상태로 표시하고, 정답 제출 시 정상 완료로 교체한다.
- 개념 요약은 Teal 왼쪽 선, 피드백은 성공·힌트·오류별 왼쪽 선과 문장을 함께 쓴다.
- HUD와 결과 지표는 작은 카드 여러 개가 아니라 하나의 행과 구분선으로 묶는다.
- FSM·지도·게임처럼 경계가 필요한 시뮬레이션만 독립된 프레임을 유지한다.
- 탭은 `aria-selected`, `aria-controls`, 방향키 이동을 지원한다.
- 진행률은 `role="progressbar"`와 현재 값을 제공한다.
- 학생 화면에는 총 수업 시간이나 미션별 분 표시를 두지 않고 현재 미션 번호와 진행률만 제공한다.

## 캐릭터 자산

- `assets/`의 SVG를 브랜드, FSM, 경로 격자, 게임 캔버스에 일관되게 사용한다.
- 플레이어는 방향 회전과 입 열림/닫힘 이미지를 번갈아 사용한다.
- Coral·Blue 고스트 정체성을 모든 보기에서 유지한다.
- 캐릭터가 필요한 곳에 이모지나 문자 대체물을 추가하지 않는다.

## 학생 행렬 지도

- 학생의 N × N Python 행렬은 원본 데이터로 유지하고 화면에서만 한 칸짜리 자동 외벽을 추가한다.
- 최종 보드 크기는 (N + 2) × (N + 2)이며, 4 × 4 입력은 6 × 6으로 렌더링한다.
- 값 0은 어두운 통로와 펠릿, 값 1과 자동 외벽은 파란 팩맨 미로 벽으로 표현한다.
- 값 2는 Pac-Man, 값 3은 고스트 위치이며 각각 정확히 한 번 사용한다.
- Map Lab은 두 원본 행렬 좌표와 Manhattan distance를 함께 표시한다.
- 지표의 벽 수는 학생 벽과 자동 외벽의 합이며 상태 문장에서 두 수를 따로 설명한다.

## 전체 Dijkstra Visual Guide

- Path Lab의 `Open guide` 버튼은 `dijkstra_visual_guide.html`을 전체화면 대화상자 안에서 연다.
- 이 파일은 제공된 원본과 SHA-256 해시가 동일한 복사본으로 유지한다.
- 원본의 7개 맵, Step/Run, 1×–8× 속도, 고스트·팩맨 배치, 비용 편집, 단계 설명을 모두 제공한다.
- 닫을 때 원본의 자동 실행 타이머를 멈춰 백그라운드 계산을 방지한다.
- 원본 파일은 변경하지 않고, 640px 이하에서만 부모 사이트가 최소 반응형 CSS를 주입해 헤더와 탭이 잘리지 않게 한다.
- `?lab=path&guide=1`로 전체 가이드를 직접 열 수 있다.

## 배포 전 점검

- 375 / 768 / 1024 / 1440 / 1920px 문서 폭 점검
- 44×44px 미만 주요 컨트롤 점검
- 한 개의 `h1`, 순차적 제목 구조, skip link 확인
- 탭 순서와 포커스 표시 확인
- FSM 슬라이더, Dijkstra Step/Run/Reset, Play/Pause/Restart 확인
- 미션 정답 채점과 Next mission 이동 확인
- 모든 JS·CSS·SVG 요청 200 확인
- 콘솔 예외와 네트워크 실패 없음 확인
